### React 프레임워크 (1강)

- 리액트는 외부라이브러리이다.
- 리액트는 코드 작성을 쉽게하기 위해 사용한다.
- 리액트는 JSX를 사용한다
    - JSX = JacvaScript XML
    - = 자바스크립트와 같지만, 자바스크립트 코드안에 HTML을 직접적으로 작성가능하다.
    - 그래서 가독성이 좋아지고 간결해진다.

```jsx
      const div = (
        <div>
          <button>hello</button>
          <p>paragraph of text</p>
        </div>
      );

      const container = document.querySelector('.js-container');
    //   ReactDOM.createRoot(container).render(button);
    //   ReactDOM.createRoot(container).render(paragraph);
      ReactDOM.createRoot(container).render(div);
```

> div라는 변수로 묶어서 한번에 렌더링 가능하다 (나중엔 컴포넌트로 묶어서 함)
> 

### React (2강)

### 컴포넌트

- 컴포넌트 첫글자는 **“대문자”**로 시작해야한다.
- PascalCase 사용
- JSX는 일반HTML보다 엄격해서 closing tag가필요하다 (모든요소에)
    - 원래 <input>만 써도 되는데,
    - **<input> </input>** 이렇게 써야한다
    - **<input/>** 은 **shortcut 버전** 위랑 동일하다.

### Fragment (<> </>)

- div 태그없이
- <>  </> 이런식으로도 사용 가능하다.
- 개발자도구로 코드를 볼때 div로 나누어져 있지않고 바로 볼 수 있어, 깔끔하다

### Placeholder

- input 입력할때 희미하게 (메시지입력하시오) 이렇게 떠있는 것을
- placeholder라고 한다.

### props

- props는 컴포넌트를 재사용하게 만들어준다.
- **props**(properties의 줄임말)는 **부모 컴포넌트에서 자식 컴포넌트로 데이터를 전달하는 방법**

```jsx
function ChatMessage(props){} 
```

- 이런식으로 사용해도 되지만,
- 이런식으로 사용하면

```jsx
const message = props.message  // 
```

- 변수선언과, props.message를 사용해야한다.

```jsx
function ChatMessage({ message, sender } ){{
```

- 그래서 이렇게 구조분해 할당해서 사용한다.
- 그러면 변수선언할 필요없이 바로 return {message} 가능

### 조건부 렌더링 (shortcut)

```jsx
        if (sender === 'robot'){
          return(
            <div>
              <img src="robot.png" width="50"/>              
              {message}
            </div>            
          )
        }
```

```jsx
{sender === 'robot' && <img src="robot.png" width="50"/>}
```

- if문 보다는 Guard Operator(&&) 많이 사용한다.
- 간결하다.
